# 卡片管理系统文档

## 概述

卡片管理系统是一个完整的后台管理解决方案，用于管理网站上的工具（tools）和游戏（games）卡片。系统提供了创建、编辑、删除、发布、隐藏等全套功能，并支持批量操作和高级搜索筛选。

## 功能特性

### 1. 卡片查看与筛选
- **列表视图**：以表格形式显示所有卡片，支持排序
- **高级筛选**：
  - 按名称/描述/标签搜索（防抖搜索，500ms延迟）
  - 按类型筛选（工具/游戏）
  - 按状态筛选（待上架/已上架/隐藏）
  - 按分类筛选（效率工具/工作流/小游戏）
- **排序功能**：支持按名称、类型、状态、创建时间排序

### 2. 卡片管理
- **创建卡片**：支持新建卡片并保存到数据库
- **编辑卡片**：编辑卡片的所有属性（名称、描述、链接、图片等）
- **删除卡片**：永久删除卡片数据
- **复制卡片**：快速复制现有卡片作为新卡片的模板
- **状态管理**：
  - draft（待上架）：管理员在编辑中
  - published（已上架）：在前台页面可见
  - hidden（隐藏）：隐藏但保留数据

### 3. 批量操作
- **批量删除**：一次删除多个卡片
- **批量发布**：将多个卡片设置为已上架状态
- **批量隐藏**：将多个卡片隐藏
- **批量复制**：一次复制多个卡片

### 4. UI 设计
- **Neo-Brutalism 风格**：
  - 硬边界（3px 边框）
  - 偏移阴影（4px 4px 0px 效果）
  - 高饱和度配色（薄荷绿、泡泡糖粉、亮黄）
  - 点击按压动画

## 数据库结构

### tools 表字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | UUID | 主键 |
| name | TEXT | 卡片名称 |
| description | TEXT | 卡片描述 |
| type | ENUM | 类型：'tool' 或 'game' |
| category | TEXT | 分类（效率工具/工作流/小游戏） |
| url | TEXT | 链接 URL |
| image_url | TEXT | 缩略图 URL |
| tags | ARRAY | 标签数组 |
| status | ENUM | 状态：'draft'/'published'/'hidden' |
| order | INT | 排序顺序 |
| featured | BOOLEAN | 是否推荐 |
| is_paid | BOOLEAN | 是否付费 |
| created_at | TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | 更新时间 |

## 前端组件架构

### 核心组件

1. **CardManagement.jsx** （主页面）
   - 整合所有子组件
   - 管理全局状态（卡片列表、选中项、筛选条件）
   - 处理 Supabase API 调用

2. **CardListView.jsx** （列表展示）
   - 表格式显示卡片
   - 支持排序、选择、单项操作
   - 显示卡片的所有关键信息

3. **FilterBar.jsx** （筛选栏）
   - 搜索输入框（防抖搜索）
   - 类型、状态、分类下拉选择
   - 活跃筛选标签显示

4. **CardEditForm.jsx** （编辑表单）
   - 卡片详细信息编辑
   - 表单验证：
     - 必填字段检查
     - 长度限制（名称50字符、描述500字符）
     - 排序值必须是非负整数
   - 字段包括：名称、描述、类型、分类、链接、图片、排序、标签、特殊标记等

5. **CardEditModal.jsx** （编辑模态框）
   - 包装 CardEditForm
   - 模态框交互处理
   - 错误信息显示

6. **ConfirmDialog.jsx** （确认对话框）
   - 删除、批量操作等危险操作的确认
   - 支持自定义文本和样式

7. **StatusToggleButtons.jsx** （状态转换按钮）
   - 显示当前状态
   - 提供可用的状态转换选项
   - 验证状态转移合法性

8. **BulkActionBar.jsx** （批量操作栏）
   - 粘性定位在页面底部
   - 显示已选择数量
   - 提供批量操作按钮

## 使用方式

### 访问卡片管理页面

```
URL: /admin/cards
权限: 需要登录
```

### 工作流示例

#### 1. 创建新卡片
- 在数据库中创建新工具（通过 API）
- 刷新页面查看新卡片
- 点击编辑修改详细信息
- 完成后发布（status 改为 published）

#### 2. 编辑现有卡片
- 在列表中找到卡片
- 点击编辑按钮（铅笔图标）
- 修改表单内容
- 点击保存

#### 3. 发布卡片
- 单项发布：点击卡片状态按钮
- 批量发布：
  1. 勾选多个卡片
  2. 批量操作栏中点击"发布"按钮

#### 4. 删除卡片
- 单项删除：点击删除按钮（垃圾桶图标）
- 批量删除：
  1. 勾选多个卡片
  2. 批量操作栏中点击"删除"按钮（红色）
  3. 确认删除

#### 5. 搜索卡片
- 在搜索框输入关键词
- 系统会在 500ms 后自动搜索
- 支持按名称、描述、标签搜索

## 前后台一体化

### 发布流程
1. 在 CardManagement 后台创建/编辑卡片
2. 将卡片状态设为 `published`
3. 前台 Tools.jsx 自动查询 `status='published'` 的卡片
4. 用户在 `/tools` 页面看到已发布的卡片

### 隐藏机制
- 卡片状态改为 `hidden` 时
- 前台页面不会显示该卡片
- 数据保留在数据库中，可以随时恢复

## 技术栈

- **前端框架**：React 18 + Vite
- **数据库**：Supabase (PostgreSQL)
- **UI 库**：
  - 自定义 Neo-Brutalism 组件
  - lucide-react（图标）
- **状态管理**：React Hooks (useState, useCallback)

## 故障排查

### 卡片不显示在前台
- 检查卡片状态是否为 `published`
- 检查数据库连接是否正常
- 查看浏览器控制台是否有错误

### 编辑不生效
- 检查表单验证是否通过
- 查看浏览器网络请求是否成功
- 检查 Supabase 权限设置

### 批量操作失败
- 确认已勾选卡片
- 查看是否有网络错误
- 尝试刷新页面重新加载
